# 🧠 DAG Training Planner — Backend

> A FastAPI backend that models running training plans as **Directed Acyclic Graphs**.  
> Every workout is a node. Every coaching rule is a directed edge.  
> Miss a session and the graph replans automatically.

---

## What This Is

Most running apps generate flat weekly schedules — a coach designs a fixed progression and everyone follows it. This backend models training plans as a proper **DAG with constraint satisfaction**, making every scheduling decision explicit, auditable, and adaptive.

```
Flat schedule (Runna today):   Mon → Tue → Wed → Thu → Fri → Sat → Sun
                                (no dependency reasoning)

DAG (this project):            EasyRun ──HARD──→ TempoRun
                                  (fresh legs required before quality session)
                               LongRun ──HARD──→ Rest
                                  (recovery required after long run)
                               TempoRun W5 ──HARD──→ TempoRun W6
                                  (progressive overload chain)
```

When an athlete misses a workout, the replanner BFS-traverses the hard constraint chain and automatically cascades skips to all transitively dependent sessions.

## Core Algorithms

### Topological Sort — Kahn's Algorithm (1962)
**Time complexity: O(V + E)**

The schedule is generated by topological sort over the DAG. Nodes with no unmet prerequisites are scheduled first. This guarantees a valid execution order where every dependency is respected.

```python
# Simplified — see core/graph.py for full implementation
def topological_sort(self) -> List[WorkoutNode]:
    in_degree = {nid: 0 for nid in self._nodes}
    for nid in self._nodes:
        for succ in self._adj[nid]:
            in_degree[succ] += 1

    queue  = deque([nid for nid, deg in in_degree.items() if deg == 0])
    result = []
    while queue:
        node_id = queue.popleft()
        result.append(self._nodes[node_id])
        for succ in self._adj[node_id]:
            in_degree[succ] -= 1
            if in_degree[succ] == 0:
                queue.append(succ)
    return result
```

### Cycle Detection — DFS
Before adding any edge, the graph checks if the new edge would create a cycle. A cycle in a training plan is logically impossible (A must come before B AND B must come before A).

### Dynamic Replanning — BFS on Hard Edges
When an athlete misses a workout, the replanner BFS-traverses only HARD edges to find all transitively affected downstream sessions.

```
Miss W5 Interval Run
  → W6 Tempo Run (HARD successor)    → SKIPPED
    → W7 Interval Run (HARD successor) → SKIPPED
       → Race dependency preserved
```

## Constraint Types

| Type | Meaning | Example |
|---|---|---|
| **HARD** | Must be satisfied — replanner acts on violation | Rest must follow Long Run |
| **SOFT** | Should be satisfied — quality score impact | Easy run preferred before Long Run |
| **LOAD** | Weekly TRIMP budget constraint | Weekly load ≤ previous week × 1.10 |

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Health check |
| `POST` | `/plans/generate` | Generate a new training plan DAG |
| `GET` | `/plans/{plan_id}` | Get current plan state |
| `GET` | `/plans/{plan_id}/week/{week}` | Get one week's workouts |
| `POST` | `/plans/{plan_id}/complete/{node_id}` | Mark workout complete |
| `POST` | `/plans/{plan_id}/miss/{node_id}` | Mark workout missed → triggers replan |
| `DELETE` | `/plans/{plan_id}` | Delete a plan |

Full interactive docs available at `/docs` (Swagger UI) and `/redoc`.

## Quick Start

```bash
# Clone and navigate to backend directory
git clone <repo-url>
cd dag-workout-planner

# Create virtual environment
python -m venv venv
source venv/bin/activate      # Mac/Linux
venv\Scripts\activate          # Windows

# Install dependencies
pip install -r requirements.txt

# Run the server
uvicorn api.main:app --reload --port 8000
```

Open `http://localhost:8000/docs` for the interactive API explorer.

## Generate a Plan — Example Request

```bash
curl -X POST http://localhost:8000/plans/generate \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Rahul",
    "experience_level": "intermediate",
    "weeks_to_race": 12,
    "race_distance_km": 42.2,
    "current_ctl": 42.0,
    "easy_run_pace_min_km": 5.5
  }'
```

Response includes:
- `nodes` — ReactFlow-ready node objects with (x, y) positions
- `edges` — ReactFlow-ready edge objects with constraint labels
- `weeks` — week-by-week summary with TRIMP loads and phase labels
- `plan_id` — use for subsequent requests

---

## Mark a Workout Missed

```bash
curl -X POST http://localhost:8000/plans/{plan_id}/miss/{node_id} \
  -H "Content-Type: application/json" \
  -d '{}'
```

Response includes:
- `affected_count` — how many nodes were affected by the replan
- `changes` — list of every node that changed and why
- `warnings` — quality sessions that were dropped
- `updated_plan` — full updated plan ready for re-render

## Plan Structure — 12-Week Marathon

```
Phase       Weeks    Focus
─────────────────────────────────────────────────
Base        1-3      Easy aerobic volume, no quality sessions
Build       4-7      Progressive overload, introduce tempo + intervals
Peak        8-10     Highest load, race-specific sessions
Taper       11-12    Reduce volume, sharpen, arrive fresh
Race        13       Marathon 🏁
```

Each phase is encoded as node durations and the quality session chain is wired as HARD edges — miss a build-phase interval and all subsequent intervals in the chain cascade to SKIPPED.
